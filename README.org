* talk-rs

Voice dictation for Linux. Record, transcribe, and paste text into any
application -- all from a single keyboard shortcut.

=talk-rs= captures audio from your microphone, sends it to a
transcription API (Mistral Voxtral or OpenAI Whisper), and types the
result into the focused window. A small X11 overlay badge shows the
current state (recording / transcribing) so you always know what is
happening.

** Features

- *Dictation workflow* -- press a key to start recording, press again
  to stop; transcribed text is pasted automatically.
- *Multiple providers* -- Mistral (Voxtral) and OpenAI (Whisper /
  GPT-4o) for batch transcription; Mistral and OpenAI realtime
  streaming via WebSocket.
- *Speaker diarization* -- identify who is speaking (~--diarize~);
  output is tagged with speaker labels.  Currently supported with
  Mistral V2 models in batch mode.
- *Multi-candidate picker* -- with ~--pick~, run several providers in
  parallel and choose the best transcription from a GTK picker window.
  A waterfall spectrogram of the recording is shown above the
  candidate list; it loads asynchronously and adapts to window width.
- *Visual overlay* -- non-intrusive X11 badge at the top of the screen
  (works without a compositor).
- *Dead audio detection* -- overlay badge shows a red prohibit icon
  and "NO SOUND" warning when no real microphone is detected (e.g.
  headset unplugged); a notification also appears on the text panel.
- *Auto-pause* -- automatically pauses audio forwarding during
  silence, trimming dead air from transcription input.  Resumes
  instantly with a 300 ms lookback buffer to preserve speech onset.
  The badge shows yellow pause bars and "LISTENING" during pauses.
  Disable with ~--no-auto-pause~.
- *Audio visualizers* -- optional in-badge visualization during
  recording (~--viz waterfall~, ~--viz amplitude~, ~--viz spectrum~);
  monochrome mode with ~--mono~.
- *Audio feedback* -- start/stop tones and periodic boop during
  recording; fully configurable or disabled.
- *Context bias* -- supply domain-specific vocabulary to improve
  transcription accuracy.
- *Daemon toggle mode* -- first invocation starts recording, second
  stops and transcribes; ideal for global shortcuts.
- *Retry last* -- re-transcribe the last cached recording without
  speaking again (~--retry-last~).
- *Recordings browser* -- ~record --ui~ opens a GTK4 window listing
  all recordings (OGG from ~output_dir~) and dictation cache (WAV).
  Play, delete, or open in the file manager; sections auto-refresh
  via inotify when files change externally.
- *Standalone commands* -- ~record~ and ~transcribe~ can be used
  independently for scripting.
- *Environment overrides* -- every config value can be set via
  ~TALK_RS_*~ environment variables.

** Prerequisites

*** Build dependencies

#+begin_src sh
# Debian / Ubuntu
sudo apt install build-essential pkg-config libasound2-dev libopus-dev \
  libgtk-4-dev libpipewire-0.3-dev libspa-0.2-dev libclang-dev \
  libssl-dev

# Fedora
sudo dnf install alsa-lib-devel opus-devel pkg-config \
  gtk4-devel pipewire-devel spa-devel clang-devel \
  openssl-devel
#+end_src

A working Rust toolchain is required (1.70+). Install via
[[https://rustup.rs][rustup]] if needed.

The optional ~whisper_local~ provider (offline transcription via
whisper.cpp with CUDA) requires the NVIDIA CUDA toolkit so that
~nvcc~ is available at build time:

#+begin_src sh
# Debian / Ubuntu
sudo apt install nvidia-cuda-toolkit
#+end_src

CPU-only builds work without the toolkit but are too slow for
practical dictation with the ~large-v3~ model.

*** Runtime dependencies

=PipeWire= must be running (used for audio capture). Most modern Linux
desktops ship with PipeWire by default.

Pick one of the transcription backends below.

**** Cloud (default)

An API key for at least one cloud provider is required:

- [[https://console.mistral.ai/][Mistral]] (default)
- [[https://platform.openai.com/][OpenAI]]

**** Local whisper.cpp (offline, GPU-accelerated)

No API key needed.  Download a GGML model file from
[[https://huggingface.co/ggerganov/whisper.cpp][ggerganov/whisper.cpp]]:

#+begin_src sh
mkdir -p ~/.local/share/talk-rs/models
curl -L -o ~/.local/share/talk-rs/models/ggml-large-v3.bin \
  https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-large-v3.bin
#+end_src

Smaller models (~medium~, ~small~, ~base~, ~tiny~) trade quality for
speed and VRAM.  ~large-v3~ needs ~3.5 GB of VRAM and gives the best
French accuracy.  See the
[[https://github.com/ggerganov/whisper.cpp/blob/master/models/README.md][whisper.cpp model README]]
for the full catalog.

** Installation

If building from a git clone, run ~./autogen.sh~ first to resolve
version placeholders in ~Cargo.toml~:

#+begin_src sh
./autogen.sh
cargo build --release
#+end_src

The binary is at ~target/release/talk-rs~. Copy it somewhere in your
~$PATH~:

#+begin_src sh
cp target/release/talk-rs ~/.local/bin/
#+end_src

** Configuration

=talk-rs= reads ~$XDG_CONFIG_HOME/talk-rs/config.yaml~ (typically
~~/.config/talk-rs/config.yaml~).

Copy the example and fill in your values:

#+begin_src sh
mkdir -p ~/.config/talk-rs
cp config.example.yaml ~/.config/talk-rs/config.yaml
#+end_src

Minimal working configuration (cloud — Mistral):

#+begin_src yaml
output_dir: ~/talk-rs-output

providers:
  mistral:
    api_key: YOUR_MISTRAL_API_KEY
#+end_src

Minimal working configuration (local — whisper.cpp, no API key):

#+begin_src yaml
output_dir: ~/talk-rs-output

providers:
  whisper_local:
    model_path: ~/.local/share/talk-rs/models/ggml-large-v3.bin
    language: fr            # or "auto"
    initial_prompt: >-
      Vocabulary hint: project names, technical terms, proper nouns…

transcription:
  default_provider: whisper_local
#+end_src

*** Required fields

| Field                                  | Description                                            |
|----------------------------------------+--------------------------------------------------------|
| ~output_dir~                           | Absolute path to a writable directory for recordings   |
| ~providers.mistral.api_key~            | Mistral API key (if using Mistral)                     |
| ~providers.openai.api_key~             | OpenAI API key (if using OpenAI)                       |
| ~providers.whisper_local.model_path~   | Path to a GGML model file (if using ~whisper_local~)   |

*** Optional fields

| Field                              | Default                   | Description                        |
|------------------------------------+---------------------------+------------------------------------|
| ~providers.mistral.url~            | ~https://api.mistral.ai~  | Mistral API base URL               |
| ~providers.mistral.model~          | ~voxtral-mini-2507~       | Mistral transcription model        |
| ~providers.mistral.context_bias~   | /none/                    | Comma-separated words for accuracy |
| ~providers.openai.url~             | ~https://api.openai.com~  | OpenAI API base URL                |
| ~providers.openai.model~           | ~whisper-1~               | OpenAI batch model                 |
| ~providers.openai.realtime_model~  | ~gpt-4o-mini-transcribe~  | OpenAI realtime model              |
| ~transcription.default_provider~   | ~mistral~                 | Default provider when unspecified  |
| ~indicators.boop_interval_ms~      | ~5000~                    | Periodic boop interval in ms (~0~ disables boops; also ~--no-boop~) |
| ~indicators.visual_overlay~        | ~true~                    | Show X11 overlay badge             |
| ~indicators.viz~                   | /none/                    | In-badge visualizer: ~waterfall~, ~amplitude~, or ~spectrum~ (also ~--viz~; env ~TALK_RS_INDICATORS_VIZ~) |
| ~indicators.mono~                  | ~false~                   | Monochrome visualizer (also ~--mono~) |
| ~paste.chunk_chars~                | ~150~                     | Max chars per paste chunk (~0~ disables chunking; also ~--no-chunk-paste~) |
| ~paste.no_paste~                   | ~false~                   | Skip auto-paste: leave the transcription in the clipboard only.  Recommended on Wayland — see "Wayland tips" below. |
| ~providers.whisper_local.language~       | ~auto~      | Language code (~fr~, ~en~, …) or ~auto~      |
| ~providers.whisper_local.use_gpu~        | ~true~      | Run inference on the GPU when built with ~cuda~ |
| ~providers.whisper_local.threads~        | ~4~         | CPU threads for non-GPU stages               |
| ~providers.whisper_local.initial_prompt~ | /none/      | Free-form vocabulary hint (local equivalent of ~context_bias~) |

*** Environment overrides

Every config value can be overridden via environment variables:

#+begin_src sh
export TALK_RS_PROVIDERS_MISTRAL_API_KEY="sk-..."
export TALK_RS_PROVIDERS_OPENAI_API_KEY="sk-..."
#+end_src

See ~config.example.yaml~ for the full list.

*** Wayland tips

talk-rs's auto-paste path uses the X11 ~XTEST~ extension to fake a
~Ctrl+Shift+V~ keystroke into the focused window.  Under a Wayland
compositor (GNOME ≥ 41, KDE Plasma, sway, …) this keystroke is
blocked: the transcription ends up in the clipboard only briefly,
then talk-rs restores the previous clipboard contents — leaving the
user with nothing to paste.

The recommended Wayland workflow is therefore:

1. Set ~paste.no_paste: true~ in the config.  In this mode talk-rs
   writes the transcription to both the X11 and Wayland clipboards
   and exits without simulating any keystroke or restoring the
   previous clipboard contents.
2. Install ~wl-clipboard~ so ~wl-copy~ is available to talk-rs:
   ~sudo apt install wl-clipboard~.
3. Install a clipboard manager so each transcription is kept in
   history and easy to insert: [[https://github.com/Keruspe/GPaste][GPaste]]
   (~sudo apt install gpaste-2 gnome-shell-extension-gpaste~),
   [[https://github.com/oae/gnome-shell-pano][Pano]] (GNOME Shell extension),
   [[https://github.com/boerdereinar/copyous][Copyous]], or any other
   manager that observes the system clipboard.
4. Trigger talk-rs as usual; press ~Ctrl+V~ (or pick from the
   clipboard manager UI) to insert the transcription where you want
   it.

** Usage

*** Global options

| Flag | Effect                                              |
|------+-----------------------------------------------------|
| ~-v~ | Increase logging verbosity (~-vv~ debug, ~-vvv~ trace) |

*** Dictate (main workflow)

Record, transcribe, and paste into the focused application:

#+begin_src sh
talk-rs dictate
#+end_src

Toggle mode (ideal for keyboard shortcuts):

#+begin_src sh
talk-rs dictate --toggle
#+end_src

First call starts a background daemon that records. Second call stops
recording, transcribes, and pastes the result.

Options:

| Flag                        | Effect                                          |
|-----------------------------+-------------------------------------------------|
| ~--toggle~                  | Daemon toggle mode                              |
| ~--provider~                | Choose ~mistral~ or ~openai~                    |
| ~--model~                   | Override model for this invocation               |
| ~--diarize~                 | Enable speaker diarization (batch mode only)    |
| ~--realtime~                | Stream audio via WebSocket (incremental text)   |
| ~--pick~                    | Show multi-candidate picker (GTK window)        |
| ~--retry-last~              | Re-transcribe the last cached recording         |
| ~--replace-last-paste~      | Delete previous paste before inserting new text |
| ~--save <PATH>~             | Save audio recording to a file                  |
| ~--output-yaml <FILE>~      | Write transcription metadata YAML               |
| ~--input-audio-file <FILE>~ | Feed a pre-recorded WAV instead of live mic     |
| ~--monitor~                 | Mix system audio (monitor) with mic input       |
| ~--no-sounds~               | Disable audio indicators                        |
| ~--no-boop~                 | Disable periodic boop sounds (keep start/stop)  |
| ~--no-chunk-paste~          | Paste all text in one shot (disable chunking)    |
| ~--no-overlay~              | Disable visual overlay                          |
| ~--no-auto-pause~           | Disable auto-pause during silence (forward all audio) |
| ~--viz <MODE>~              | In-badge visualizer: ~waterfall~, ~amplitude~, or ~spectrum~ |
| ~--mono~                    | Monochrome visualizer (theme-aware)             |

*** Record

Capture audio to an OGG/Opus file:

#+begin_src sh
talk-rs record                       # auto-named memo-YYYY-MM-DD-HH-MM-SS.ogg
talk-rs record meeting-notes.ogg     # custom filename
#+end_src

Options:

| Flag        | Effect                                           |
|-------------+--------------------------------------------------|
| ~--monitor~ | Mix system audio (monitor) with microphone input |
| ~--ui~      | Open GTK4 recordings browser (play, delete, open folder) |

*** Transcribe

Transcribe an existing audio file:

#+begin_src sh
talk-rs transcribe recording.ogg                # print to stdout
talk-rs transcribe recording.ogg output.txt     # write to file
talk-rs transcribe recording.ogg --provider openai
#+end_src

Options:

| Flag         | Effect                                       |
|--------------+----------------------------------------------|
| ~--provider~ | Choose ~mistral~ or ~openai~                 |
| ~--model~    | Override model for this invocation            |
| ~--diarize~  | Enable speaker diarization (tag by speaker)  |

** GNOME keyboard shortcut

Bind ~talk-rs dictate --toggle~ to a key (e.g. =Super+/=):

#+begin_src sh
BASE="org.gnome.settings-daemon.plugins.media-keys"
BPATH="/org/gnome/settings-daemon/plugins/media-keys/custom-keybindings"

CURRENT=$(gsettings get "$BASE" custom-keybindings)
N=0
while echo "$CURRENT" | grep -q "custom${N}/"; do
  N=$((N + 1))
done
SLOT="${BPATH}/custom${N}/"
SCHEMA="${BASE}.custom-keybinding:${SLOT}"

if [ "$CURRENT" = "@as []" ]; then
  gsettings set "$BASE" custom-keybindings "['${SLOT}']"
else
  gsettings set "$BASE" custom-keybindings "$(echo "$CURRENT" | sed "s|]$|, '${SLOT}']|")"
fi

gsettings set "$SCHEMA" name    'talk-rs dictate'
gsettings set "$SCHEMA" command 'talk-rs dictate --toggle --viz waterfall'
gsettings set "$SCHEMA" binding '<Super>slash'
#+end_src

First press starts recording, second press stops, transcribes, and
pastes into the focused application.

To change the key, replace ~<Super>slash~ with the desired binding
(e.g. ~<Super>semicolon~, ~<Super>d~). Add ~--realtime~ to use
streaming transcription instead of batch mode.
